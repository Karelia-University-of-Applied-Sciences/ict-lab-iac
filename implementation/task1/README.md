# Task1: Multi-User Nutanix VM Provisioning with Cloud-Init

## Overview

Task1 provisions Ubuntu 24.04.2 VMs on Nutanix clusters with automated user management via cloud-init. The configuration is designed for **iterating over a list of users**, where each user gets their own dedicated VM.

The workflow is project-first:
1. OpenTofu creates or manages the Nutanix project.
2. OpenTofu then creates one VM per user in that project.

**Key Features:**
- Terraform-managed project lifecycle before VM creation
- One VM per user (e.g., neo, morpheus, trinity → 3 VMs)
- Automatic Ubuntu user creation (admin) with password or SSH keys
- Support for shared admin users (ansible, deploy, etc.)
- Regular user accounts with optional sudo permissions
- All configuration applied on first VM boot via cloud-init

## Quick Start

### 1. Define Users

Edit `infra.auto.tfvars`:

```hcl
default_subnet_uuid = "<SUBNET_UUID>"

users = [
  { name = "neo", password = "neo_pass_123", sudo = true },
  { name = "morpheus", password = "morpheus_pass_456" },
  { name = "trinity", password = "trinity_pass_789" },
]
```

### 2. Configure Ubuntu Password

```hcl
ubuntu_password = "shared_ubuntu_password"
```

### 3. Plan and Apply

```bash
tofu plan
tofu apply
```

Result: 3 VMs created
- `alex-tf-task1-vm-neo`
- `alex-tf-task1-vm-morpheus`
- `alex-tf-task1-vm-trinity`

## Configuration

### Basic Users List

Simplest form - just usernames and passwords:

```hcl
users = [
  { name = "alice", password = "alice_pass" },
  { name = "bob", password = "bob_pass" },
]
```

Creates 2 VMs, each with:
- ubuntu user (admin) with shared `ubuntu_password`
- User account (alice/bob) with their password
- No sudo access for alice or bob

### Adding Sudo Permission

Grant passwordless sudo to specific users:

```hcl
users = [
  { name = "alice", password = "alice_pass", sudo = true },
  { name = "bob", password = "bob_pass" },
]
```

Alice can run `sudo` without entering a password. Bob cannot.

### SSH Keys for Ubuntu User

Use SSH authentication instead of password:

```hcl
ubuntu_ssh_keys = [
  "ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAAB...",
  "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI...",
]
ubuntu_password = ""  # Set empty if using SSH keys only
```

All VMs will have ubuntu user accessible via SSH keys. Password is ignored.

### Shared Admin Users

Add shared admin accounts (ansible, deployment users) to all VMs:

```hcl
admin_users = [
  {
    name     = "ansible"
    ssh_keys = ["ssh-rsa AAAAB3NzaC1yc2EA..."]
  },
  {
    name     = "deploy"
    password = "deploy_user_password_123"
  },
]
```

Every VM will have both `ansible` (SSH access) and `deploy` (password access) users with passwordless sudo.

### SSH Keys for Regular Users

Give users SSH access instead of passwords:

```hcl
users = [
  {
    name     = "alice"
    ssh_keys = ["ssh-rsa AAAAB3NzaC1yc2EA...", "ssh-ed25519 AAAAC3NzaC1..."]
    sudo     = true
  },
  {
    name     = "bob"
    password = "bob_password_fallback"
  },
]
```

Alice logs in via SSH keys. Bob uses password. Both have sudo.

## Complete Example

```hcl
# ubuntu password (shared across all VMs)
ubuntu_password = "UbuntuP@ssw0rd123"

# Optional: shared admin users on all VMs
admin_users = [
  {
    name     = "ansible"
    ssh_keys = [
      "ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAAB...",
    ]
  },
  {
    name     = "deploy"
    password = "DeployP@ss123"
  },
]

# Users: each gets one VM
users = [
  {
    name     = "neo"
    password = "NeoP@ss123"
    sudo     = true
  },
  {
    name     = "morpheus"
    password = "MorpheusP@ss456"
    sudo     = false
  },
  {
    name     = "trinity"
    ssh_keys = [
      "ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAAB...",
    ]
    sudo = true
  },
]

# VM naming
vm_name_prefix = "ai-vm"  # Results in: ai-vm-neo, ai-vm-morpheus, ai-vm-trinity
```

## VM User Structure

Each VM created has this user structure:

```
Ubuntu User (Admin)
├─ Username: ubuntu
├─ Access: Password or SSH keys (configurable)
└─ Sudo: YES (passwordless)

Admin Users (Shared, Optional)
├─ Username: ansible, deploy, etc.
├─ Access: Password or SSH keys
└─ Sudo: YES (passwordless)

User-Specific Account
├─ Username: From users list (neo, morpheus, trinity)
├─ Access: Password or SSH keys
└─ Sudo: Optional (per-user setting)
```

## How Cloud-Init Works

1. **Nutanix applies cloud-config** during VM first boot via guest customization
2. **Cloud-init processes YAML** to create users and apply configuration
3. **Users created** with specified passwords, SSH keys, and permissions
4. **Terraform module** generates this YAML automatically from variables

The YAML structure (auto-generated by module):

```yaml
#cloud-config
users:
  - name: ubuntu
    password: 'UbuntuP@ssw0rd123'
    sudo: ALL=(ALL) NOPASSWD:ALL
  - name: ansible
    ssh_authorized_keys:
      - ssh-rsa AAAAB3NzaC1yc2EA...
    sudo: ALL=(ALL) NOPASSWD:ALL
  - name: neo
    password: 'NeoP@ss123'
    sudo: ALL=(ALL) NOPASSWD:ALL
  - name: morpheus
    password: 'MorpheusP@ss456'
```

## Variable Reference

### ubuntu_password

Password for ubuntu admin user on all VMs.

```hcl
ubuntu_password = "my_password_123"
```

Leave empty if using `ubuntu_ssh_keys` only:
```hcl
ubuntu_password = ""
```

### ubuntu_ssh_keys

SSH public keys for ubuntu user (alternative to password).

```hcl
ubuntu_ssh_keys = [
  "ssh-rsa AAAAB3NzaC1yc2EA...",
  "ssh-ed25519 AAAAC3NzaC1...",
]
```

### admin_users

Shared admin accounts on all VMs. Optional.

```hcl
admin_users = [
  {
    name     = "ansible"
    password = "ans_pass"      # Optional
    ssh_keys = ["ssh-rsa ..."] # Optional
  },
]
```

Each admin user must have either `password` or `ssh_keys` (or both).

### users

List of regular users. Each user gets one VM.

```hcl
users = [
  {
    name     = "alice"
    password = "alice_pass"    # Optional
    ssh_keys = ["ssh-rsa ..."] # Optional
    sudo     = true            # Optional, default false
  },
]
```

Each user must have either `password` or `ssh_keys`.

### vm_name_prefix

Prefix for VM names. VMs named as: `{prefix}-{username}`

```hcl
vm_name_prefix = "prod-vm"  # Results in: prod-vm-alice, prod-vm-bob
```

## Troubleshooting

### VMs created but can't login

1. **Check cloud-init logs** on VM:
   ```bash
   sudo cat /var/log/cloud-init-output.log
   ```

2. **Verify username/password** matches `users` list in `infra.auto.tfvars`

3. **Check SSH keys** are correctly formatted (single-line strings)

### Password not working but SSH keys configured

When `ubuntu_ssh_keys` is provided, password is ignored for ubuntu user. Use SSH key only.

### One user failed, others succeeded

Cloud-init is per-VM. If one user's VM fails, others may still succeed. Check the failed VM's cloud-init logs.

## Module Architecture

This configuration uses two modules.

### nutanix_project module

Located at `../terraform_modules/nutanix_project/`.

The module:
- **Creates or manages the Nutanix project** before any VMs are provisioned
- **Assigns the cluster and subnet** to the project via `cluster_uuid` and `default_subnet_uuid`
- **Accepts an optional description** via `project_description` (defaults to `project_name`)

`module "vms"` depends on this module via `depends_on` to ensure the project exists before VMs are created.

### nutanix_vm module

Located at `../terraform_modules/nutanix_vm/`.

The module:
- **Generates cloud-init YAML** from variables using the `cloud-init.tftpl` template
- **Creates SCSI boot disk** from `disk_size_gb`
- **Attaches Ubuntu image** for OS installation
- **Configures network** via specified subnet
- **Applies guest customization** with generated users

Changes to user management happen at the **module level**, not in task1.

## Notes

- All user passwords are passed in plain text via Terraform variables. Use environment variables or secrets manager for sensitive deployments.
- SSH keys are preferred over passwords for security.
- Cloud-init runs on first boot only. Changes to `infra.auto.tfvars` after apply do not affect already-created VMs.
- If the project already exists outside Terraform, import it into state before apply to avoid duplicate-name creation errors:
  - `tofu import module.project.nutanix_project.this <PROJECT_UUID>`
